    Introduction and purpose

    Multiplex PCR involves many challenges (primer design & validation, concentrations of primers, polymerases, salts (monovalent & divalent) and dNTPS, temperatures, durations and the number of primers and targets in one reaction) and problems (formation of unwanted products due to unspecific primer design or targets close to each other, and primer dimer formation by cross-dimerization, self-dimerization or hairpin formation). During evaluation and validation of multiplex primer and probe sets, these problems are often observed but not always explainable. However due to the ease of getting whole genome sequences, we have developped PRIMEval for an in silico evaluation of multiplex primer and probe sets.

    PRIMEval maps primers and probes to whole genomes or contigs by a background method which can be chosen by the user (Bowtie 1.2.2, BLAST+ 2.7.1 or the Aho–Corasick string-searching algorithm). The user can set a maximum number of allowed mismatches in primers and probes and a maximum amplicon length generated by two primers. The mapped primers and probes which meet the mismatch criteria are shown as 'hits'. If primer hits occur within a certain distance as defined by the maximum amplicon length, a possible amplicon is reported as 'result' and probes which bind to an amplicon are also shown. Not all observed or missing products in the wet lab can be explained by this in silico specificity check. Therefore we have implemented a secondary structure check using the Primer3 2.3.7 ntthal application which checks primer and probe sets for cross-dimers, self-dimers and hairpins depending on the reaction conditions (monovalent & divalent cations, dNTPs, DNA concentration). Problematic oligonucleotides are reported using deltaG and TM values and represented in heatmaps.

    The antibiotic resistance gene and virulence factor identifier (ARVI) is a practical implementation of PRIMEval. ARVI uses a defined set of primers and probes designed against 405 antibiotic resistance genes and 405 virulence factors. Shortly, more than 4000 resistance genes were downloaded from the CARD [1] and ARG-ANNOT [2] databases and virulence factor genes were downloaded from the VFDB [3]. Sequences were clustered with 90% sequence similarity in a one-against-all approach and the representative sequence of each cluster was the one with the most BLAST hits. For both antibiotic resistance genes and virulence factors, the 405 gene clusters with the most hits were used for primer and probe design. Primer and probe design was done using Oli2go in 9 different sets each [1]. These primers and probes are used in PRIMEval for screening purposes. Due to an additional implementation of mapping files in PRIMEval, the user can compare the in silico products to the intended products and show additional information (e.g. for antibiotic resistance genes: related to these resistances, resistance mechanism, antibiotic, AMRGeneFamily; for virulence factors: gene related to, from organism). The data was scraped from the CARD or VFDB, respectively.

    1. Jia, Baofeng, et al. "CARD 2017: expansion and model-centric curation of the comprehensive antibiotic resistance database." Nucleic acids research (2016): gkw1004.
    2. Gupta, Sushim Kumar, et al. "ARG-ANNOT, a new bioinformatic tool to discover antibiotic resistance genes in bacterial genomes." Antimicrobial agents and chemotherapy 58.1 (2014): 212-220.
    3. Chen, Lihong, et al. "VFDB 2016: hierarchical and refined dataset for big data analysis—10 years on." Nucleic acids research 44.D1 (2015): D694-D697.
    4. Hendling, Michaela, et al. "Oli2go: an automated multiplex oligonucleotide design tool." Nucleic acids research (2018).

    Workflow

    Input & Parameters

    Help for all input parameters is shown as popup on the submission page. However, some parameters will be explained here in detail:

    • Method: Bowtie (default), BLAST or Aho-Corasick. The advantages of each method are described in detail in our publication. Shortly, Aho-Corasick is the recommended & fastest method for a low number of contigs and a low number of mismatches (0 or 1) if no wobbled bases are used in the primers and probes (otherwise a too high number of mismatches might be reported due to the nature of the algorithm). For a higher number of mismatches, Aho-Corasick becomes incredibly slow. Bowtie is the recommended method for any number of mismatches if wobbled primers or probes are used. If screening with a maximum number of 3 mismatches is desired, BLAST performs faster than Bowtie however the number of hits and results is significantly reduced.
    • Input files: Each primer or probe set has to be uploaded as FASTA file with the file ending .fasta. Each primer file should contain all forward and reverse primers from that set and each probe file should contain all probes from that set. The file name corresponds to the set name and the corresponding probe set needs to have the extension _probe.fasta. For example, if the primer set is named abr1.fasta, the corresponding probe set file needs to be named abr1_probes.fasta. The contigs for each strain or sample should be uploaded as one FASTA file with the file ending *.fasta; also the file name represents the sample or strain name (e.g. E_coli_14E.fasta). Mapping files are uploaded as CSV file. While the first 5 columns cannot be renamed and have to stay in the same order (Primer1;Primer2;Probe;Gene;ExpectedSize), an unlimited number of columns can be added to the right. It is best to download our sample data and have a look at the included CSV files.
    • Cross-check primer packages: By default, primers and probes are only matched if the come from the same set. This condition is met if the file names correspond to each other (e.g. package1.fasta and package1_probes.fasta) as described above.
    • Check probes only: This box needs to be checked if no primers are uploaded and only probes are mapped to the uploaded contigs. If primers are uploaded, they will be discarded.
    • Secondary structure check: The secondary structure check performs a cross-dimer, self-dimer and hairpin check on each uploaded oligonucleotide set. For best results, the user needs to adjust parameters for monovalent & divalent cations, dNTPS and DNA concentration.

    Output

    Following sections are available in PRIMEval for finished runs:

    • Browse: A visual representation of forward and reverse primers with the amplicon they form. Probes binding to the amplicon are also represented. This serves as a quick overview for the screening of problematic oligos.
    • Results: All products of forward and reverse primers and eventually matching probes are listed. Mismatches are indicated, and whether primers and probes come from the same or different oligo sets and if more probes potentially bind to the same amplicon.
    • Hits: A list of all primers and probes binding to the uploaded contigs, independently whether they form products in PCR.
    • Maps: (Optional) This section maps the amplicons to the user-defined mapping files.
    • Dimer Check: (Optional) The dimer check shows potentially conflicting oligos by identifying cross-dimers, self-dimers (primers) and hairpins (primers & probes).
    • Parameters: A summary table of anonymized input parameters (e.g. excluding email address).
    • Download: A table containing links to download all output files with (additional) complete information.

    Acknowledgments

    The PRIMEval software is based on three components: (1) the PRIMEval client application (oligonucleotide mapping and secondary structure check tool), (2) the PRIMEval server application which handles the communication between the PRIMEval client application and the webserver application, and (3) the webserver application which allows public access to PRIMEval.

    The PRIMEval client application is written in Python 3.4.3. The underlying oligonucleotide mapping algorithms are based on BLAST+ 2.7.1 [1,2], Bowtie 1.2.2 [3] and the Python package pyahocorasick 1.1.8. The secondary structure check uses the ntthal application from Primer3 2.3.7 [4]. Also following third party Python software packages are used: Biopython 1.69 [5], pandas 0.22 [6], NumPy 1.8.2 [7] and XlsxWriter 0.9.3.

    The PRIMEval server application is written in Python 3.4.3. It uses the Redis 2.8.4 server application and the Python third party software packages redis 2.10.6 and rq 0.12 (Redis Queue). Also following third party Python software packages are used for file and information exchange: urllib3 1.7.1 and PyYAML 3.13.

    The webserver application is implemented in Python 3.6.7 using the Django 2.1.4 web framework running on an Apache2 webserver and a PostgreSQL database. The client side makes use of following frameworks and libraries: Bootstrap, jQuery, igv.js [8], Font Awesome, DropzoneJS, Popper.js and Google Analytics. The server-side uses following third party Python software packages: matplotlib 3.0.2 [9], seaborn 0.9, NumPy 1.15.4, pandas 0.23.4, urllib3 1.24.1, PyYAML 3.13 and python-magic 0.4.15.
